# Chrono Reverse Implementation Log

このファイルは、Chrono Reverse のこれまでの実装記録と、今後の作業ログをまとめる場所です。新しい実装・UI調整・アセット生成・仕様変更・検証結果が発生したら追記します。

## 更新ルール

- 作業日、目的、変更ファイル、実装内容、検証結果、残課題を記録する。
- Imagegen でアセットを作った場合は、用途、保存先、プロンプト要約、UI反映先を記録する。
- 既存の未実装項目を解消した場合は、`docs/まだ未実装.md` の該当範囲も確認する。

## 現在の全体像

最終整理日: 2026-05-16

Chrono Reverse は、時間進行・3レーン・MP回復・自動攻撃・Active Response・EXポケット・墓地・ヒーローアーツを持つ 1vs1 カードバトルとして実装されている。Web は Next.js、ゲームコアは TypeScript、オンライン対戦は Node.js + WebSocket、モバイル版は Expo/React Native をベースにしている。

## 実装済み領域

### Game Core

- `core/types.ts` でゲーム状態、プレイヤー、カード、ユニット、イベント、入力の型を定義。
- `core/engine.ts` を中心に、ターンレスな時間進行、MP回復、ユニット攻撃、レーン交戦、直接攻撃を実装。
- `core/activeResponse.ts`、`utils/activeResponseMp.ts`、`components/ActiveResponse*` 系で Active Response の解決・MP補正・表示を実装。
- `core/cardPlayFlow.ts`、`core/cardTargeting.ts` でカードプレイ時の対象選択・配置・発動フローを分離。
- `core/effects.ts`、`core/newCardEffects.ts`、`core/cardSpecificEffects.ts`、`core/effectParser.ts` でカード効果の実装・パース・個別処理を管理。
- `core/heroAbilities.ts`、`core/heroes.ts` でヒーローとヒーローアーツを管理。
- `core/csvLoader.ts` と `public/新カードCore - カードデータのスプレッドシート化.csv` でカードデータを読み込む構成を追加。
- `tests/core/` に Active Response、効果、カード対象選択、新カード効果のテストを追加。

### Web UI

- `pages/index.tsx`、`pages/title.module.css` でタイトル画面を実装。
- `components/AppShell.tsx`、`components/ScreenRouter.tsx`、`components/BottomNavigation*.tsx` でページ遷移からコンポーネント方式へ寄せたアプリシェルを実装。
- `components/screens/HomeScreen.tsx`、`CardsScreen.tsx`、`DeckListScreen.tsx`、`DeckEditScreen.tsx`、`DeckSelectScreen.tsx`、`DeckViewScreen.tsx`、`MatchmakingScreen.tsx` を実装。
- `components/GameBoard.tsx`、`components/OnlineGameBoard.tsx`、`pages/battle.tsx`、`pages/battle.module.css` でバトル画面を実装。
- スマホ横画面対応、縦画面ロック表示、スクロール禁止、矢印・レーン・カード重なりの調整を実施。
- カード詳細モーダル、カード一覧、デッキ編集、デッキ一覧、デッキ選択、ホーム画面を本家風の構成に近づけて調整。
- `HeroModel3D.tsx`、`HeroPortrait.tsx` でヒーロー表示、3Dモデル表示を実装。

### Online / Server

- `server/gameServer.ts`、`server/index.ts`、`server/matchmaking.ts` で WebSocket ベースのオンライン対戦とマッチメイキングを実装。
- `core/protocol.ts` と `utils/useGameSocket.ts` でクライアント・サーバー間の通信型と接続フックを管理。
- ゲーム開始時刻同期、オンライン対戦時のタイマー、Active Response まわりの同期調整を実施。
- Render デプロイ関連の設定と `server/allowedOrigins.ts`、`tests/server/allowedOrigins.test.ts` を追加。

### React Native

- `native/` に Expo ベースの React Native アプリ基盤を追加。
- `native/src/app/AppShell.tsx`、`native/src/app/navigation.tsx` でネイティブ側の画面構成を実装。
- `native/src/screens/MainScreens.tsx`、`BattleScreen.tsx` で主要画面とバトル画面を実装。
- `native/src/hooks/useCards.ts`、`usePracticeBattle.ts`、`useNativeGameSocket.ts` でカード読み込み、練習バトル、通信を実装。
- `native/src/storage/decks.ts` でデッキ保存を実装。
- `native/src/components/HeroModel3D.tsx`、`native/src/assets.ts` でネイティブ側のアセット参照を整理。

### Cards / Effects

- 重貫通、臨戦、俊敏、空戦、リベンジ、MPブースト、シールド、目覚め、成長、撃破、プレイ時効果、action_effect、振り分けダメージなどを段階的に実装。

実装内容:

- 8ヒーロー分のLive2D向け立ち絵をImagegenで生成し、クロマキー背景を透明化してプロジェクト内に保存。
- `Hero.live2dImageUrl` と将来のCubism差し替え用 `Hero.live2dModelUrl` を追加。
- Webのホーム、デッキ一覧、デッキ選択、バトル画面のヒーロー表示を `HeroLive2D` へ差し替え。
- 生成PNGをベースに、呼吸、上半身揺れ、髪/マント揺れ、視線追従、瞬き風グロー、属性パーティクルを重ねた疑似Live2Dモーションを実装。
- React Native側にも `HeroLive2D` を追加し、ホーム、デッキ一覧、デッキ選択、バトル画面で同じImagegenアセットを使うようにした。

プロンプト要約:

- 各ヒーロー名、属性、性格、衣装、武器、立ち姿を指定し、カードバトルゲーム向けの高品質アニメ調フルボディ立ち絵として生成。
- UI組み込み前提のため、文字なし、ロゴなし、UI枠なし、全身、十分な余白、クロマキー背景、影なしを指定。

検証:

- `npm run type-check`: 通過。
- `cd native && npm run type-check`: 通過。
- `git diff --check`: 通過。
- `npm run build`: Turbopackがサンドボックス内でプロセス/ポート作成できず失敗。TypeScript段階は通過していたが、最適化フェーズで `TurbopackInternalError: creating new process`。
- `npm run dev`: 既存ポート競合のため3001で起動表示。ただしこの実行環境から `curl` で `127.0.0.1:3001` へ接続できず、HTTP確認は未完了。

残課題:

- 本物のLive2D Cubismとして動かすには、`.model3.json`、`.moc3`、テクスチャ、物理演算設定、モーション定義をLive2D Cubism Editor等で作成して `live2dModelUrl` に接続する必要がある。現状はImagegen立ち絵を使った疑似Live2Dモーション。
